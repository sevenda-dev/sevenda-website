How it works

Three steps to a
BPMN diagram

From raw browser events to structured process models — without lifting a pen.

01

Record

Click start in DevTools. Sevenda captures clicks, form submissions, navigation events, network calls, and DOM mutations — everything that matters.

clicks navigation API calls DOM mutations
02

AI generates

Claude Sonnet reads the event stream, identifies the process flow, and produces validated BPMN 2.0 XML with correct pools, lanes, gateways, and sequence flows.

Claude Sonnet 4 BPMN 2.0
03

Export

Download .bpmn for Camunda, Signavio or Bizagi. Export .svg or .png for presentations. Generate a full process report in .docx format — ready to share with stakeholders, no technical skills required. Refine with natural language chat.

USE CASES

Built for the people who document processes.

Three teams. Three problems. One tool.

Business Analyst

AS-IS process documentation. Day one.

A new client engagement starts Monday. No process documentation exists. The BA opens Chrome DevTools, navigates through the client's web application for 90 seconds, and presses Generate. Sevenda produces a validated BPMN 2.0 diagram — with every task, gateway, and flow captured automatically.

  • ✓BPMN 2.0 diagram ready for Camunda or Signavio
  • ✓Analisi Tecnico-Funzionale .docx for stakeholders
  • ✓Jira issue created with diagram attached

From 8 hours → 90 seconds

Marketing & Data Consultant

GTM audit. Full gap report. One session.

The client's GA4 is configured, but conversion events are missing from half the funnel. The consultant switches Sevenda to Insights mode, navigates the purchase flow, and clicks Generate. Every untracked interaction is flagged — with CSS selectors, dataLayer.push() snippets, and a ready-to-import Tag Plan.

  • ✓Tracking gap analysis with element selectors
  • ✓dataLayer.push() snippets ready for GTM
  • ✓Tag Plan JSON — importable directly into GTM
  • ✓Analytics Report .docx for the client

From 3-day audit → 20 minutes

Developer / QA

Ship fast. Document automatically.

Release day. No time to update the process docs. The developer records the QA session in Sevenda — every API call, every navigation, every form submission captured in the event stream. One click generates the updated BPMN and a structured report, ready to commit alongside the code.

  • ✓User flow diagram — always up to date
  • ✓API interaction log with status codes and timing
  • ✓Export .svg for README or Confluence
  • ✓Natural language refinement for edge cases

Features

Built for process professionals

Real-time event capture

Captures clicks, form submissions, API calls, navigation events, and DOM mutations with zero configuration — works on any website.

Claude Sonnet AI

Anthropic's most capable model understands context, identifies decision points, and produces accurate process models — not just a sequence of steps.

BPMN 2.0 standard

Output is fully compliant BPMN 2.0 XML. Open directly in Camunda Modeler, Signavio, Bizagi, or any standards-compliant tool.

Iterative refinement

Chat with the diagram. "Add an error path after the API call" and Sevenda updates the BPMN while preserving the rest of the model exactly.

Marketing & Funnel Mapping

Visualize customer journeys, campaign flows, and conversion funnels automatically — no diagramming tools, no manual work, no technical skills required.

Multi-format export

.bpmn for process engines, .svg and .png for presentations, .docx reports for stakeholders. One session, every format you need.
